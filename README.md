# designsystemet — Brand Token Repo

This repo contains the **design token source of truth** for a theme built on top of [Digdir Designsystemet](https://designsystemet.no). It does **not** contain component code — components come from the npm packages `@digdir/designsystemet-css` and `@digdir/designsystemet-web`.

---

## What lives here

| Path | What it is |
|---|---|
| `designsystemet.config.json` | **Source of truth.** Defines brand colours, border-radius, and typography. Edit this to change your theme. |
| `design-tokens/` | Generated token JSON files (do not edit directly — they are regenerated by `tokens create`). Used by Token Studio / Figma. |
| `design-tokens-build/` | Generated CSS custom properties and TypeScript types (do not edit directly — regenerated by `tokens build`). Used by your app at runtime. |
| `demo/` | A Vite demo site that renders Designsystemet components styled with your tokens. |

---

## How the token pipeline works

```
designsystemet.config.json
        │
        ├─ tokens create ──▶  design-tokens/*.json       (Token Studio / Figma sync)
        │
        └─ tokens build  ──▶  design-tokens-build/
                               ├── theme.css             (CSS custom properties for your app)
                               ├── types.d.ts
                               └── colors.d.ts
```

Components from `@digdir/designsystemet-css` reference CSS variables like `--ds-color-accent-base-default`. Your `theme.css` defines the values of those variables based on your config. See [`designsystemet-explained.md`](designsystemet-explained.md) for the full deep-dive.

## Changing the theme

1. Edit [`designsystemet.config.json`](designsystemet.config.json) — change colours, border-radius, or typography.
2. Regenerate tokens:
   ```bash
   npx @digdir/designsystemet@latest tokens create --config designsystemet.config.json
   ```
3. Rebuild CSS and types:
   ```bash
   npx @digdir/designsystemet@latest tokens build --config designsystemet.config.json
   ```
4. Commit and push. Designers pull the updated tokens in Token Studio → export to Figma variables.

> **Warning:** Never edit files inside `design-tokens/` or `design-tokens-build/` directly — they are fully overwritten on every run.

## Demo site

A Vite-based demo site lives in `demo/`. It imports `@digdir/designsystemet-css` for component styles and the built `theme.css` for your token values, and renders every major component so you can visually verify your theme works end-to-end.

### First-time setup

```bash
cd demo
npm install
```

### Run the dev server

```bash
cd demo
npm run dev
```

Opens at **http://localhost:5173** (or the next available port).

### Build for production

```bash
cd demo
npm run build
```

Output goes to `demo/dist/`.

### Preview the production build

```bash
cd demo
npm run preview
```

### What the demo shows

- Color palette swatches for all token color scales (accent, neutral, support colors)
- Buttons — all variants (primary / secondary / tertiary), colors, sizes, and disabled states
- Typography — full heading and body text scale
- Form components — text input, textarea, select, checkbox, radio, switch, validation errors
- Alerts — info, success, warning, danger
- Cards, tags, divider, spinner
- Size modes — small / medium / large side-by-side
- **Dark / light mode toggle** — switches `data-color-scheme` on `<html>` to verify both color schemes
